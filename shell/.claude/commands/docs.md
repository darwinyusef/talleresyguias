# Comando: /docs

Genera o actualiza la documentación del proyecto.

## Argumentos:
- `type` (opcional): api, readme, all (default: all)

## Pasos:

1. Analiza la estructura del proyecto
2. Identifica código sin documentar o con documentación desactualizada
3. Genera documentación apropiada:
   - README.md con descripción del proyecto
   - Comentarios de código para funciones/clases
   - Documentación de API si aplica
   - Guías de uso y ejemplos
4. Actualiza archivos existentes o crea nuevos
5. Muestra un resumen de lo que se documentó
